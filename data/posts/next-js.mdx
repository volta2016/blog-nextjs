# ¿ Qué es Next JS ? 🌟

Next js es un framework creado por Guillermo Rauch, que nos permite crear fácilmente sitios web y aplicaciones web de React.

## Guillermo Rauch.

Guillermo Co-fundador de Vercel ▲ creador de socket.io y Next JS.

## Vercel y Next JS

**Vercel y Next js** nació de la búsqueda de una manera mucho más rápida y mucho más intuitiva de hacer app con react todos saben que CRA no es la solución definitiva y hay muchas cosas que debemos hacer optimizaciones de fuentes css e imágenes.

Con Next js tienes SEO, ruta, llamada a servicios tienes todo eso muchísimo más en un solo lugar, no tienes que ocuparte de hacer todas esa optimizaciones tediosas para poder lograr un buen performance.

## Next js te da el control total de lo que tu puedes hacer con tu app:

- <span className="list-icon">☑️ </span> Tu aplicaron puede ser híbrida
- <span className="list-icon">☑️ </span> Puede ser client side only
- <span className="list-icon">☑️ </span> No es necesario usar las 3 juntas
- <span className="list-icon">☑️ </span> Vas usar unas rutas simples con algunos
  pedidos a una API y aprovecharemos API root, pero todo siempre va ser client side

## ¿Para qué y por qué se usa NextJS?

![uso next js](/static/images/uso-next-js.png "uso-next-js")

### Beneficios de usar Next-JS:

- <span className="list-icon">☑️ </span> Fast Refresh ▶️ recarga rápida

- <span className="list-icon">☑️ </span>File System Routing ▶️ rutas desde el
  nombre de los archivos.

- <span className="list-icon">☑️ </span> API Routes ▶️ que es para el backend{" "}
- <span className="list-icon">☑️ </span> Build in css support > Soporte para sass
  y less

- <span className="list-icon">☑️ </span> Facilidad para aprender y sin complejas
  configuraciones.
- <span className="list-icon">☑️ </span> Optimizado para brindar mejoras de SEO y
  rendimiento.
- <span className="list-icon">☑️ </span> Enrutamiento basado en páginas con soporte
  de rutas dinámicas.
- <span className="list-icon">☑️ </span> Rutas API desplegadas en funciones Serverless.
- <span className="list-icon">☑️ </span> Capacidad para generar sitios estáticos
  (SSG), usar server-side rendering (SSR) o una combinación de ambos según la necesidad
  de cada página.
- <span className="list-icon">☑️ </span> Compatibilidad incorporada con CSS y SASS.
  Además de soporte para múltiples librerías de CSS en JS.
